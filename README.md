# NitsQC — проверка яркости шота в нитах (Nuke, ACES/HDR)

Гизмо для Nuke, которое показывает **абсолютную яркость шота в нитах (cd/m²)** и проверяет,
попадает ли она в допустимый диапазон (по умолчанию **0–650 нит**) — для самоконтроля перед ОТК.

---

## Зачем это нужно

Значения `ACEScg`/`ACES2065-1` — это **относительный scene-linear свет**, у них нет нитов сами по себе.
Абсолютная яркость (ниты) появляется только после **ACES Output Transform (ODT)** на конкретный
HDR-дисплей (PQ / ST.2084). Гизмо делает именно это:

```
вход (ACEScg или ACES2065-1)  ->  ACES ODT (Rec.2020 ST2084 1000 нит)  ->  PQ-декод  ->  НИТЫ
```

Это стандартная референс-проверка, как в HDR-скопах Resolve/Nuke.

---

## Установка

1. Скопируй `NitsQC.gizmo` в папку `~/.nuke/` (или в любую папку из `NUKE_PATH`):
   ```
   cp NitsQC.gizmo ~/.nuke/
   ```
2. (Опционально) чтобы появился пункт меню — добавь в `~/.nuke/menu.py`:
   ```python
   nuke.menu('Nodes').addMenu('QC').addCommand('NitsQC', 'nuke.createNode("NitsQC")')
   ```
3. Перезапусти Nuke (или, если только скопировал .gizmo — можно сразу `Tab` → `NitsQC`).

**Требование:** проект в OCIO-режиме с ACES-конфигом (гизмо берёт активный OCIO из переменной `$OCIO`).
На этой машине: `OCIO=/opt/ocio/OpenColorIO-Config-ACES-1.2/aces_1.2/config.ocio`.

---

## Быстрый старт

1. `Read` (твой EXR) → **NitsQC**.
2. Проверь поле **`Vhodnoe prostranstvo`** (см. ниже — ACEScg или ACES2065-1).
3. Нажми **`ZAMERIT KADR`**.
4. Смотри **`MAKS. YARKOST, nit`** и **`Status`** (в диапазоне / превышение). Число дублируется на лейбле ноды.
5. Для визуала переключи **`Rezhim`** → `False color`: красным подсветит зоны выше лимита.

---

## Что подавать на вход: ACEScg vs ACES2065-1

Гизмо конвертирует из пространства, указанного в поле **`Vhodnoe prostranstvo`**.
Оно должно совпадать с тем, в каком пространстве **пиксели на входе ноды** (рабочее пространство комп-графа).

| Ситуация | Что ставить в `Vhodnoe prostranstvo` |
|---|---|
| **Проект в ACEScg** (рабочее пространство `scene_linear = ACES - ACEScg`, дефолт aces_1.2) | `ACES - ACEScg` *(значение по умолчанию — ничего менять не надо)* |
| **Данные на входе в ACES2065-1** (AP0) — читаешь ACES2065-1 без конверсии / рабочее пространство AP0 | `ACES - ACES2065-1` |

Правило простое: **`Vhodnoe prostranstvo` = то пространство, в котором реально лежат числа перед гизмо.**
Обычно в OCIO-комп-графе это рабочее пространство (для aces_1.2 — ACEScg). Если явно гонишь ACES2065-1 —
переключи на `ACES - ACES2065-1`.

> Ниты на выходе для одного и того же снятого света **одинаковы** при любом корректно указанном входе —
> ODT приводит оба к одному дисплею. Важно лишь не соврать про входное пространство.

## Описание параметров

| Параметр | Назначение |
|---|---|
| **`Vhodnoe prostranstvo`** | Входной colorspace (ACEScg по умолч. / ACES2065-1). Должен совпадать с пространством пикселей на входе. |
| **`HDR output (ODT)`** | Выходное преобразование ACES на HDR-дисплей. По умолч. `Output - Rec.2020 ST2084 (1000 nits)`. От него зависят ниты — ставь то, в чём меряет ОТК (Rec.2020 / P3-D65, пик 1000/2000/4000). |
| **`Metrika`** | Как считать «яркость»: **Max R/G/B (MaxCLL)** — максимум из каналов, стандарт HDR-деливери, самый строгий (по умолч.); **Yarkost Y** — перцептивная светлота (Rec.2020 веса). |
| **`Limit (nit)`** | Порог ОТК. По умолч. **650**. |
| **`Porog predupr (dolya limita)`** | Доля лимита для жёлтой зоны предупреждения. `0.9` = жёлтый начинается с 585 нит. |
| **`Rezhim (viewer)`** | Что показывать в вьюпорте: **False color** (🟢 OK / 🟡 близко / 🔴 превышение) или **Znachenie v nitah** (r=g=b=ниты, для точного замера пипеткой). |
| **`ZAMERIT KADR`** | Кнопка: считает **максимум по всему кадру** и пишет результат. |
| **`MAKS. YARKOST, nit`** | Результат — макс. яркость кадра в нитах. |
| **`Status`** | Вердикт: «в диапазоне (запас N нит)» или «ПРЕВЫШЕНИЕ на N нит». Дублируется на лейбле ноды. |

## Эталонные файлы (папка `reference/`)

Готовые EXR (ACEScg) с известными значениями — загрузи, прогони гизмо, сверь число.
Таблица — **фактические замеры самой гизмой** (ODT `Rec.2020 ST2084 1000 nits`, метрика `Max R/G/B`):

| Файл | Вход ACEScg | Ожидаемо, нит | Смысл |
|---|---|---|---|
| `ref_01_grey18.exr` | 0.18 | **15.0** | mid-grey 18% (якорь ACES) |
| `ref_02_white.exr` | 1.0 | **107.4** | diffuse white (якорь ACES) |
| `ref_03_val5.exr` | 5.0 | **361.9** | средние света |
| `ref_04_nearlimit.exr` | 18.0 | **645.0** | впритык под лимитом 650 |
| `ref_05_over.exr` | 100.0 | **891.4** | заведомо выше лимита (проверка красной зоны) |
| `ref_06_ramp_0-30.exr` | 0 → 30 (градиент) | до **737.9** | плавный переход, видно где пересекает 650 (в False color) |

Как проверять: `Read` файла (colorspace `ACES - ACEScg`) → `NitsQC` → `ZAMERIT KADR`.
Число в `MAKS. YARKOST, nit` должно совпасть со столбцом «Ожидаемо».

## Сверка с мировыми эталонами (валидация математики)

Гизмо сверено с опубликованными эталонными значениями — совпадение практически идеальное.

### PQ-декодер vs SMPTE ST.2084 (нормированный код → нит)

| Код | Эталон, нит | Гизмо | Δ |
|---|---|---|---|
| 0.5081 | 100 | 100.0 | 0.0% |
| 0.6963 | 600 | 600.0 | 0.0% |
| 0.7518 | 1000 | 999.8 | 0.0% |
| 0.9026 | 4000 | 4001 | 0.0% |
| 1.0 | 10000 | 10000 | 0.0% |

### ACES ODT якоря vs эталон ACES (ST2084 P3-D65 1000 нит)

| Вход ACEScg | Эталон, нит | Гизмо | Δ |
|---|---|---|---|
| 0.18 (mid-grey) | 14.51 | 15.0 | +3.4%* |
| 1.0 (diffuse white) | 106.56 | 107.4 | +0.7% |

\* Эталон 14.51/106.56 — из спецификации **ACES 2.0**, а конфиг машины — **ACES 1.2**.
У версий чуть разная кривая в тенях, отсюда расхождение именно на mid-grey. Для ACES 1.2 значения гизмы корректны.

Источники: SMPTE ST.2084, BT.2100 (PQ); ACES Output Transforms (ampas/aces-core).

## Согласование с ОТК

Софт ОТК неизвестен, поэтому это **референсная проверка по стандарту ACES/PQ**, а не 1-в-1 их инструмент.
Если узнаешь их метод — подгони под него два поля:

- **`HDR output (ODT)`** — то же выходное пространство (Rec.2020 или P3-D65, тот же пик по нитам).
- **`Metrika`** — если ОТК меряет пиковую светимость (MaxCLL) → `Max R/G/B`; если перцептивную яркость → `Yarkost Y`.

Большинство HDR-QC работают в PQ/ST.2084 — тогда гизмо совпадёт близко.
